# Notion 페이지 나눔 미리보기 (Notion PDF Preview)

Notion 페이지를 **내보내기(PDF) 전에** A4~A1 용지 기준으로 어느 줄이 몇 페이지에 들어가는지 눈으로 확인하고, **형광펜·네모·텍스트 주석**을 얹어 바로 PDF로 저장하는 크롬 확장(MV3)입니다. 미리보기 = 출력(WYSIWYG).

## 왜?

Notion의 기본 PDF 내보내기는 서버에서 렌더링되어, 내보내기 전엔 페이지 배치를 알 수 없습니다. PDF를 열어봐야 확인되니 "내보내기 → 열기 → 수정"을 반복하게 됩니다. 이 확장은 그 피드백 루프를 없애고, 편집 후 버튼 한 번으로 페이지 경계를 미리 봅니다.

## 설치 (개발자 모드)

1. `chrome://extensions` 접속 → 우측 상단 **개발자 모드** 켜기
2. **압축해제된 확장 프로그램을 로드** → 이 폴더 선택
3. 툴바에서 확장을 고정(핀)해두면 편리

## 사용

1. `app.notion.com`에서 대상 페이지 열기
2. 확장 툴바 버튼 클릭 → 새 탭에 **페이지 나눔 미리보기**가 열림
3. 상단에서 **용지(A4~A1)**·**여백(상하/좌우)** 조절 → 즉시 재계산
4. **PDF로 저장** → 그 미리보기 그대로 출력
   - 인쇄 대화상자에서 여백/머리글이 남으면 **여백: 없음, 머리글/바닥글: 해제**

좌측 **썸네일 레일**·우측 **목차(헤딩) 레일**로 페이지·섹션을 빠르게 탐색합니다. 각 레일 가장자리 **핸들**은 **드래그=너비 조절**, **클릭=접기/펴기**입니다. 현재 페이지는 상단바 정중앙 알약(`n / N 쪽`)에 표시됩니다.

### 주석 (출력 전 강조)

미리보기에 강조를 얹어 PDF에 함께 출력합니다(도구 아이콘에 마우스를 올리면 색·두께·크기 패널이 뜸):

- **형광펜**(Alt+1): 텍스트 드래그로 강조, 강조를 다시 클릭하면 제거
- **네모박스**(Alt+2): 드래그로 빨간 테두리(이미지 등). 클릭 선택 후 이동·모서리 크기조절·`Del` 삭제
- **텍스트**(Alt+3): 클릭해 입력, 더블클릭 재편집. 이동·색·크기·`Del`
- 5색 팔레트(검정·흰색·빨강·코랄·파랑), 네모/텍스트 위 **휠로 두께·크기 조절**, `Esc`/`Ctrl+Enter`로 편집 종료
- ⚠ 용지·여백을 바꿔 **재계산하면 주석은 지워집니다** — 주석은 마지막에 다세요.

## 동작 원리

- **자체 페이지 분할기**: 블록을 mm 단위 A4~A1 페이지 박스에 순서대로 흘려 담고, 그 페이지 div를 그대로 인쇄 → 미리보기 = 출력.
- **표/큰 블록 행 단위 분할 (v2)**: 한 페이지보다 큰 표는 **행(`<tr>`) 단위로 나눠** 여러 페이지에 이어 담고, 이어지는 페이지마다 헤더(`<thead>`)를 반복합니다. 표 외 컨테이너(리스트·컬럼 등)는 자식 요소 단위로 분할. 페이지보다 큰 분할 가능 블록은 **현재 페이지 남은 공간부터** 채웁니다.
- **표 폭 맞춤 (v2)**: Notion 표는 컬럼이 고정 px라 여백이 크면 오른쪽이 잘립니다. 표를 본문 폭에 맞게 축소(`width:100%` + 셀 `min-width:0` + 자동 줄바꿈)해 여백과 무관하게 항상 페이지 안에 들어오게 합니다(Notion export와 동일한 처리).
- **빈 레이어 스킵 (v2)**: Notion 편집 UI 잔재(높이 0 블록 핸들 오버레이, 표와 같은 높이의 내용 없는 배경/선택 오버레이 레이어 등)는 페이지를 차지하지 않게 건너뜁니다(빈 페이지·표 뒤 공백 방지).
- **DB 편집 크롬 제거 (v3)**: 데이터베이스(컬렉션 뷰)의 편집용 UI를 스냅샷 단계에서 걷어내 **순수 데이터 표**만 남깁니다(Notion export와 동일). 제거 대상: 제목 옆 툴바 아이콘(필터·정렬·검색 등)·파란 "새로 만들기" 버튼·열 헤더 끝의 `+`/`...`·맨 아래 "새 페이지" 행. 헤더 끝 버튼은 안정적 클래스가 없어 "헤더 셀을 품지 않은 헤더로우 직접 자식"만 골라 제거(데이터 헤더는 보존). **표(테이블) 뷰 기준**입니다.
- **이미지 인라인**: Notion 이미지는 인증 프록시→프리사인 S3로 리다이렉트되어 확장 페이지에서 직접 로드가 막힙니다. 백그라운드 서비스워커(host 권한으로 CORS 우회)가 fetch해 `data:` URL로 박제합니다.
- **이모지 인라인 (v4)**: Notion 인라인 이모지(✅ 등)는 `<img>`의 inline style `background:url(/images/emoji/…webp)` **상대경로** 스프라이트로 그려집니다. 상대경로라 미리보기(`chrome-extension://` 출처)에선 확장 폴더 기준으로 해석돼 404 → 화면·PDF 모두 빈 칸이 됩니다. 스프라이트는 notion.so 동일 출처라 스냅샷 단계에서 직접 fetch해 `data:` URL로 박제하고, 인쇄 시엔 `print-color-adjust:exact`로 배경까지 출력합니다.
- **썸네일 레일 (v5)**: 좌측 여백에 각 페이지의 축소 클론(썸네일)을 세로로 나열해 빠른 탐색·이동을 돕습니다. 클릭 시 해당 페이지로 이동(가까우면 부드럽게, 멀면 즉시 점프), 현재 보이는 페이지는 파란 테두리로 강조(IntersectionObserver). 클론은 이미지가 `data:` URL로 인라인돼 있어 오프라인으로 완전히 렌더됩니다. 레일 가장자리 핸들로 **너비 조절(드래그)·접기(클릭)** 하며(썸네일도 너비 따라 커짐), 인쇄(PDF)에는 포함되지 않습니다.
- **목차 레일·주석·용지 확장 (v6)**: 우측에 헤딩(H1~H3) **목차 레일**(클릭 이동·현재 섹션 강조, 좌우 레일 모두 너비 드래그 조절). **형광펜·네모·텍스트 주석**을 미리보기에 얹어 PDF에 함께 출력합니다 — 주석은 각 페이지에 앵커링(네모는 `position:relative` 페이지 기준 절대좌표)되고, 테두리는 원래 인쇄되며 배경색은 `print-color-adjust:exact`로 함께 출력됩니다. 용지 **A2·A1** 추가(`@page`를 명시적 mm로 지정 — A2/A1은 CSS 명명 크기에 없어 `size:A2`가 무시됨). 현재 페이지는 하단 중앙 알약으로, 창 세로 스크롤바는 오버레이(hover 전용) 대신 **상시 표시**.
- **스타일 재현**: Notion 자체 CSS를 스냅샷해 미리보기에 적용, 인쇄용 `@page`만 우리가 통제.

## 한계

- 아주 긴 **단일 텍스트 블록**(쪼갤 요소 자식이 없는 문단 등)이 한 페이지를 넘으면 잘림(`⚠` 표시).
- **DB(데이터베이스/컬렉션 뷰)**: 편집용 UI는 걷어내지만(v3), 대응 범위는 **표(테이블) 뷰**입니다. 보드·갤러리·캘린더 등 다른 뷰는 편집 크롬 구조가 달라 아직 정리되지 않을 수 있습니다. 또한 **열이 아주 많은 DB**는 데이터 그리드가 페이지 폭보다 커서 오른쪽 열이 잘릴 수 있습니다(Notion export도 동일한 한계).
- 표 한 행 자체가 한 페이지보다 크면 그 행은 잘림.
- 대상은 **Notion 웹(app.notion.com)**. 데스크톱(Electron) 앱은 범위 밖.
- 짧은 시간에 반복 실행하면 Notion 서버가 레이트리밋(429)할 수 있습니다(잠시 후 해제).

## 파일 구성

| 파일 | 역할 |
|---|---|
| `manifest.json` | MV3 매니페스트 (권한·액션) |
| `background.js` | 서비스워커: 버튼 클릭 시 스냅샷 주입, 이미지 fetch(CORS 우회) |
| `content-snapshot.js` | Notion 페이지에서 본문+CSS 스냅샷, 이미지 인라인 위임 |
| `preview.html` / `preview.js` | 미리보기 UI + 자체 페이지 분할기 + 인쇄 |
